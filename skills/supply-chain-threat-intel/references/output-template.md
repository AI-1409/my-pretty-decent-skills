# OUTPUT TEMPLATE — Final Report Structure

## How to Present Final Findings

After aggregation and consensus filtering, present each incident using this template.

## Report Header

```markdown
# Supply Chain Attack Investigation Results

**Query:** npm crypto ecosystem in the last 6 months

**Multi-Agent Verification:** Using 4 parallel agents, aggregated with consensus filtering

---

## Confidence Grades

✅ HIGH CONFIDENCE ([N] incidents) — 80-100% consensus (4/4 agents) + primary source
⚠️  MEDIUM CONFIDENCE ([N] incidents) — 60-79% consensus (3/4 agents)
🔍 SUSPECTED ([N] incidents) — 40-59% consensus + strong source pending manual verification
❌ EXCLUDED ([N] findings) — <40% consensus or single-agent finding (see Appendix)

---

## HIGH CONFIDENCE INCIDENTS

```

## Markdown Formatting Rules

- Use `**bold**` for headers and emphasis
- Use `>` for blockquotes (primary source excerpts)
- Use code blocks for IOCs and code snippets
- Use bullet lists for sources, IOCs, tactics
- Use horizontal rules `---` to separate incidents
- Keep each incident under ~50 lines for readability

## Don't Include

❌ Single-source Reddit/HN posts without verification
❌ GitHub user accusations (@user is malicious) without evidence
❌ Unconfirmed rumors from blogs with no technical detail
❌ Findings with no CVE/GHSA or vendor acknowledgment

## Do Include

✅ CVE/GHSA + 2+ independent sources
✅ SUSPECTED findings with clear "needs verification" label
✅ Conflicts listing all claims (don't pick a "winner")
✅ All excluded findings in appendix with reasons

## Template Summary

**Main report:** HIGH → MEDIUM → SUSPECTED → CONFLICTS

**Appendix:** Excluded findings (single-agent, low consensus)

**Metadata footer:** Report generation info, configuration, agent count
